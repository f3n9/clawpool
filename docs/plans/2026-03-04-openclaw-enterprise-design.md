# OpenClaw 企业内部多实例部署设计（按需启动 + 固定实例）

## 1. 目标与约束
- 统一入口 URL，员工经 Keycloak 认证后访问自己的 OpenClaw WebGUI。
- 每个员工固定一个实例，重复登录始终进入同一实例。
- 数据持久化且严格隔离（员工 A 不可访问员工 B 数据）。
- 70 员工规模，高峰活跃约 40，支持按需启动、空闲休眠。
- 模型统一 OpenAI GPT-5.2 / GPT-5.3-codex。
- 部署目标：单机 Debian 13（6C/12T, 64GB RAM, 500GB SSD），优先 Docker。

## 2. 推荐架构
- 入口层：Traefik（或 Nginx）
- 认证层：oauth2-proxy + Keycloak OIDC
- 会话编排层：instance-manager（轻量服务）
- 业务层：每员工 1 个 OpenClaw 容器（固定容器名）
- 存储层：每员工独立 volume 与配置目录

核心链路：
1) 用户访问 `openclaw.company.internal`。
2) oauth2-proxy 触发 Keycloak 登录。
3) 认证成功后提取 `employee_id`（或 `sub`）作为用户唯一键。
4) instance-manager 根据映射表定位固定实例 `openclaw-<employee_id>`。
5) 实例未运行时自动 `docker start`，健康检查通过后路由到实例。

## 3. 实例与数据隔离策略
- 每实例独立容器、独立 volume、独立配置目录：
  - `/srv/openclaw/users/<employee_id>/data`
  - `/srv/openclaw/users/<employee_id>/config`
- 每实例 API Key 独立管理：
  - 首次启动可注入统一默认 Key（仅用于初始化）。
  - 管理员后续可将该实例替换为员工专属 Key。
  - Key 存放于员工专属 secret/config 文件，不写入镜像层。
- 容器运行用户非 root，目录权限 `700`，属主限定为实例 UID。
- 网络层仅暴露反向代理，不暴露实例端口到外网。
- 禁止容器间互访（独立网络或最小化网络策略）。

## 4. 资源与容量控制
- 初始建议（按实例）：`cpus=0.8`，`mem_limit=1.2g`。
- 允许动态资源配额：
  - 低并发时提高活跃实例上限（例如 `cpus=1.5`、`mem_limit=2g`）。
  - 并发升高时回收到基线配额，避免全机内存争抢与 OOM。
- 空闲休眠：30 分钟无活动自动 stop。
- 冷启动容忍：5-20 秒（已确认可接受）。
- 节流与保护：
  - 限制同时启动实例数（如 3-5 个）避免启动风暴。
  - 设置实例最大磁盘占用告警与日志轮转。

## 5. 通道配置与外部系统
- 用户在 WebGUI 内配置 Telegram/Google Chat/企业微信等 channel。
- 若 channel 需要凭据，存储在该用户实例专属密钥空间。
- 企业微信与 Google Workspace 集成按“每用户独立授权令牌”原则处理。
- 用户可自行修改 OpenClaw 内配置（Channels、Skills、Plugins 等）。
- 所有用户级配置与插件状态需落盘到该员工持久化目录，保证重启后可恢复。

## 6. 安全与运维
- Keycloak token 仅用于身份与授权，不直接用于业务数据存储。
- 审计日志记录：登录、实例启动/停止、配置变更。
- 备份策略：按用户目录增量备份（每日）+ 周期恢复演练。
- 升级策略：先灰度 5 用户，再全量滚动升级。
- 升级必须采用“镜像与数据分离”：
  - 升级 OpenClaw 版本时仅替换镜像与容器，不替换用户 volume。
  - 若涉及配置结构变更，提供可回滚迁移脚本并在灰度阶段先验证。

## 7. 配置与密钥生命周期
- 首次开通员工实例：
  - 预创建 `data/config` 目录并写入默认模板配置。
  - 使用统一默认 API Key 启动实例并通过健康检查。
- 运营切换到员工专属 API Key：
  - 在员工专属 secret 中更新 Key。
  - 重启该员工容器使新 Key 生效。
  - 可配置巡检任务，识别仍在使用默认 Key 的实例并告警。
- 失效与轮换：
  - 支持按员工粒度轮换，不影响其他员工实例。
  - 轮换失败仅回滚单用户实例配置，不触发全局回滚。

## 8. 非目标（当前阶段）
- 不引入 Kubernetes/K3s（当前规模与团队成本不匹配）。
- 不做多机高可用（先保证单机稳定与可观测）。
